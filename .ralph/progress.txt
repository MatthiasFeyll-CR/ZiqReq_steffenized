- Learnings for future iterations:
  - Gateway mirror models are `managed = False` — they need their own RunSQL migration for table creation (see services/gateway/apps/projects/migrations/0004_requirements_documents.py pattern).
  - PYTHONPATH in test container puts gateway first, so `apps.projects.models` always resolves to gateway's models. Core's `validate_structure` can't be imported directly in tests — must define inline or import from a shared module.
  - The `--reuse-db` pytest option means migrations must use `CREATE TABLE IF NOT EXISTS` to avoid errors on repeated runs.
  - Structure JSONB is a list of dicts. Mutations require `list(draft.structure)` to create a mutable copy before modifying, then reassigning back to `draft.structure` for Django to detect the change.
  - Two pre-existing test failures (from US-001) send `first_message` without required `project_type`. Future iterations should fix: services/gateway/apps/projects/tests/test_views.py::test_create_project_with_first_message and services/gateway/apps/collaboration/tests/test_visibility.py::test_new_project_defaults_to_private.
