- Tests: 666 python passed (25 new, 3 pre-existing failures), 365 node passed (7 new). Typecheck clean. Ruff clean.
- Learnings for future iterations:
  - Test container runs under gateway.settings.test — AI service apps (apps.context, apps.embedding) are NOT importable. Use gateway mirror models (apps.admin_ai_context.models) for model field tests.
  - Gateway mirror models are `managed = False` — migrations must use RunPython with raw cursor SQL, not RunSQL with multi-statement strings. PostgreSQL doesn't support `ADD CONSTRAINT IF NOT EXISTS` in ALTER TABLE.
  - context_chunks table is created by AI service embedding app migration, not gateway. Gateway migration must check `information_schema.tables` before altering context_chunks.
  - context_assembler.get_facilitator_context() imports FacilitatorContextBucket lazily. Must try AI service path first, fall back to gateway mirror. Also must handle DB access errors gracefully (SynchronousOnlyOperation in async tests).
  - sys.modules patching with `patch.dict(sys.modules, {...})` + `importlib.reload` is needed to mock AI service models in context_retrieval tests since the import path doesn't exist in gateway context.
  - Pre-existing test failures (3): test_new_project_defaults_to_private, test_create_project_with_first_message (from US-001), test_chunks_passed_to_sk_and_ids_returned (infrastructure issue).
