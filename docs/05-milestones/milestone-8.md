## Story Outline (Suggested Order)

1. **[Backend] Review action APIs** — Gateway REST: POST accept (no comment required), POST reject (mandatory comment), POST drop (mandatory comment), POST undo (mandatory comment, valid from accepted/dropped/rejected → returns to in_review). Core gRPC servicers: validate state transitions, update idea.state, create review_timeline_entries (entry_type='state_change'). Publish idea.review.* events. Conflict of interest: reviewer_id != owner_id and != co_owner_id.
2. **[Backend] Review assignment APIs** — Gateway REST: POST /api/reviews/:id/assign (self-assign), POST unassign. Validation: reviewer role required, cannot assign to own idea. Core gRPC: create/update review_assignments. GET /api/reviews: list ideas categorized (assigned to me, unassigned, accepted, rejected, dropped) with search by title/UUID.
3. **[Backend] Review timeline APIs** — Gateway REST: GET /api/ideas/:id/review/timeline (paginated, ordered by created_at), POST (add comment, entry_type='comment'). Nested replies: parent_entry_id references parent comment. Timeline entries are immutable (append-only). State changes and resubmissions auto-generated by system.
4. **[Frontend] Review section (below fold)** — Full viewport height below brainstorming section. Top area (always visible): small BRD preview (latest version), idea title, assigned reviewer(s), current state label. Only visible after idea has been submitted at least once (has_been_submitted check).
5. **[Frontend] Review timeline** — Chronological feed containing: comments (with nested replies), inline state change entries (styled as system text), resubmission entries (with linked document versions — both old and new as clickable download links). Comment input with submit button. Reply button on comments.
6. **[Frontend] Review page** — /reviews route, Reviewer role only. Categorized lists: Assigned to me, Unassigned, Accepted (collapsed), Rejected (collapsed), Dropped (collapsed). Search by title or UUID. Self-assign button on unassigned ideas. Unassign button on assigned ideas. Click idea → navigate to /idea/:uuid.
7. **[Frontend] Review actions** — Accept, Reject, Drop buttons on review section. Reject and Drop require mandatory comment (modal or inline). Undo button (on accepted/dropped/rejected ideas) requires mandatory comment. Multiple reviewers: any single reviewer acts independently. Latest action wins for conflicts. All actions recorded in timeline.
8. **[Frontend] Auto-scroll on state transition** — Page auto-scrolls based on idea state: Open/Rejected → brainstorming (top), In Review/Accepted/Dropped → review (bottom). Triggers on: submit, state change event via WebSocket, entering an idea page. Smooth scroll animation.
